Meet the Docs Reader skin

A reader-first layout that keeps your portfolio narrative, audio HUD, and PDF hooks intact while presenting human-friendly documentation.

Why Docs Reader?

The Docs Reader skin gives performers, commissioners, and students a familiar documentation environment. It keeps your works data, cues, and score integrations intact while presenting prose, callouts, and code samples with copy affordances.

Use the left navigation to jump between high-level areas, skim the outline on the right, and lean on instant search to surface topics. Every heading exposes a copyable deep link so you can share specific guidance with collaborators.

prae generate --skin docs-reader
prae preview --skin docs-reader

Layout anatomy

Docs Reader ships with a responsive three-pane layout. On large screens you get a persistent navigation rail, the main reading column, and a contextual outline. On mobile the panes collapse into drawers that are accessible via the menu buttons in the header.

Navigation rail

Groups collapse to keep things tidy. The component remembers open sections via localStorage so returning visitors land in a familiar state.

Outline pane

The outline reflects <h2> and <h3> headings that are visible in the viewport. An IntersectionObserver tracks the current section and highlights matching entries in both the outline and navigation.

Search & shortcuts

Start typing in the search box to see instant suggestions powered by a mocked adapter. Use / to focus search from anywhere, ↑/↓ to walk results, and Enter to open the highlighted hit.

While reading, hover a heading to reveal its link icon. Pressing c while a code sample or copy button is focused copies the block to your clipboard.

import { praeApplyTheme } from 'praetorius/theme';

const theme = praeCurrentTheme();
praeApplyTheme(theme, { persist: false });

Integrating with Works Console

The skin keeps the #works-console host in the DOM so the audio HUD and PDF features can bind without modifications. If your project uses additional console hooks, they will continue to initialize safely.

Tip: you can run prae generate --skin docs-reader --minify to emit a production bundle while keeping theme toggles and copy affordances.
